# 超级马里奥游戏 - 技术架构文档

## 1. 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                      游戏主程序                              │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐          │
│  │  游戏引擎   │  │  渲染系统   │  │  碰撞检测   │          │
│  │  (循环/状态) │  │  (Canvas 2D)│  │  (AABB)     │          │
│  └─────────────┘  └─────────────┘  └─────────────┘          │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐          │
│  │  角色系统   │  │  地图系统   │  │  物品系统   │          │
│  │  (马里奥)   │  │  (关卡数据) │  │  (金币)     │          │
│  └─────────────┘  └─────────────┘  └─────────────┘          │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐          │
│  │  敌人系统   │  │  音效系统   │  │  UI系统     │          │
│  │  (AI巡逻)   │  │  (音效播放) │  │  (HUD/菜单) │          │
│  └─────────────┘  └─────────────┘  └─────────────┘          │
└─────────────────────────────────────────────────────────────┘
```

## 2. 技术选型

- **前端框架**：原生 JavaScript + HTML5 Canvas
- **构建工具**：无需构建，直接使用单个HTML文件
- **渲染技术**：Canvas 2D API
- **游戏循环**：requestAnimationFrame
- **碰撞检测**：AABB (Axis-Aligned Bounding Box)
- **音频处理**：Web Audio API

## 3. 核心模块设计

### 3.1 游戏引擎 (Game Engine)
```
职责：
- 管理游戏主循环 (60 FPS)
- 处理游戏状态切换
- 管理所有游戏对象生命周期
- 协调各子系统更新和渲染

核心方法：
- init(): 初始化游戏
- update(): 更新所有游戏对象
- render(): 渲染所有游戏对象
- gameLoop(): 主循环入口
```

### 3.2 角色系统 (Character System)
```
职责：
- 马里奥的移动和跳跃
- 角色状态管理（站立、跑步、跳跃）
- 动画帧控制

核心属性：
- x, y: 位置坐标
- vx, vy: 速度向量
- width, height: 碰撞体积
- isJumping: 跳跃状态
- direction: 移动方向
```

### 3.3 地图系统 (Map System)
```
职责：
- 加载和渲染关卡数据
- 管理地面、砖块、管道等静态元素
- 处理视口滚动

关卡数据结构：
- platforms[]: 平台数组
- bricks[]: 砖块数组
- pipes[]: 管道数组
- enemies[]: 敌人初始位置
- coins[]: 金币位置
- flagpole: 终点位置
```

### 3.4 碰撞检测系统 (Collision System)
```
职责：
- AABB碰撞检测
- 检测角色与地面的碰撞（防止穿墙）
- 检测角色与敌人的碰撞
- 检测角色与金币的碰撞

碰撞类型：
- TOP: 从上方碰撞（落在平台上）
- BOTTOM: 从下方碰撞（头顶撞砖块）
- LEFT: 从左边碰撞
- RIGHT: 从右边碰撞
```

### 3.5 敌人系统 (Enemy System)
```
职责：
- 敌人AI巡逻行为
- 敌人移动和边界检测
- 敌人与马里奥的碰撞处理

巡逻逻辑：
- 敌人沿固定路径左右移动
- 碰到边界或障碍物时反向
```

### 3.6 UI系统 (UI System)
```
职责：
- HUD显示（分数、生命、金币）
- 开始界面
- 游戏结束界面
- 暂停菜单

UI组件：
- ScoreDisplay: 分数显示
- LifeDisplay: 生命显示
- GameOverlay: 遮罩层
- Button: 按钮组件
```

## 4. 数据结构

### 4.1 游戏对象基类
```javascript
GameObject {
  x: number,      // X坐标
  y: number,      // Y坐标
  width: number,  // 宽度
  height: number, // 高度
  active: boolean // 是否激活
}
```

### 4.2 马里奥对象
```javascript
Mario extends GameObject {
  vx: number,           // 水平速度
  vy: number,           // 垂直速度
  isJumping: boolean,   // 是否跳跃中
  direction: string,    // 面向方向
  lives: number,        // 生命数量
  score: number,        // 分数
  coinCount: number,    // 金币数量
  animationFrame: number // 动画帧
}
```

### 4.3 关卡数据格式
```javascript
Level {
  width: number,        // 关卡宽度
  height: number,       // 关卡高度
  ground: Array,        // 地面数据
  platforms: Array,     // 平台数据
  enemies: Array,       // 敌人初始位置
  coins: Array,         // 金币位置
  flagpole: Object      // 终点旗杆
}
```

```
                │
                ▼

```

## 6. 输入控制

| 按键 | 功能 |
|------|------|
| ← 左方向键 | 向左移动 |
| → 右方向键 | 向右移动 |
| 空格键 / ↑ 上方向键 | 跳跃 |
| P 键 | 暂停/继续 |
| R 键 | 重新开始 |

## 7. 性能优化

1. **对象池**：敌人和金币使用对象池复用
2. **视口裁剪**：只渲染可视区域内的对象
3. **离屏渲染**：预渲染静态背景到离屏画布
4. **节流更新**：非关键UI不每帧更新
